# ---------------------- GIT ----------------------

Git é um sistema de controle de versão distribuído.

Ele é utilizado para registrar e controlar as alterações
realizadas em arquivos de um projeto ao longo do tempo.

Permite:
- acompanhar o histórico de mudanças
- restaurar versões anteriores
- facilitar o trabalho em equipe
- organizar o desenvolvimento de software


# -------------------- GITHUB ---------------------

GitHub é uma plataforma online baseada no sistema Git.

Ela é utilizada para armazenar, compartilhar e colaborar
em projetos de software através da internet.

No GitHub é possível:
- guardar projetos em repositórios
- acompanhar o histórico de mudanças
- trabalhar em equipe
- compartilhar código com outros desenvolvedores


# ------------------- REPOSITÓRIO -----------------

Um repositório é o local onde ficam armazenados
todos os arquivos de um projeto juntamente com
o histórico de alterações feitas com Git.

Funções principais:
- armazenar os arquivos do projeto
- registrar todas as versões criadas
- permitir colaboração entre desenvolvedores
- manter o histórico completo do projeto


# -------------- CONCEITOS PRINCIPAIS --------------

----------------------- COMMIT ---------------------

Definição:
Registro de uma alteração realizada no projeto.
Cada commit salva uma nova versão no histórico.

Analogia:
Funciona como salvar diferentes versões de um
documento, permitindo voltar para versões anteriores.


------------------------ PUSH ----------------------

Definição:
Comando utilizado para enviar os commits do
computador local para o repositório remoto.

Analogia:
Semelhante a enviar a versão atualizada de um
trabalho para um espaço compartilhado do grupo.


------------------------ PULL ----------------------

Definição:
Comando utilizado para baixar atualizações do
repositório remoto para o computador.

Analogia:
É como baixar a versão mais recente de um
trabalho que um colega atualizou.


----------------------- BRANCH ---------------------

Definição:
Uma ramificação do projeto principal usada
para desenvolver novas funcionalidades ou
realizar testes sem alterar a versão principal.

# =====================================================
#            EXEMPLO DE ALGORITMO DO DIA A DIA
# =====================================================


╔════════════════════════════════════════════════════╗
║              EXEMPLO: FAZER UMA XÍCARA DE CAFÉ     ║
╚════════════════════════════════════════════════════╝


-------------------- OBJETIVO -----------------------

Preparar uma xícara de café utilizando uma sequência
lógica e organizada de passos.


---------------- ENTRADA DE DADOS -------------------

- Água
- Pó de café
- Filtro de papel
- Suporte do filtro
- Xícara
- Fonte de calor (fogão ou cafeteira)


---------------- PROCESSAMENTO ----------------------

Passo 1 -> Colocar água em uma chaleira ou recipiente.

Passo 2 -> Aquecer a água até atingir o ponto de fervura.

Passo 3 -> Posicionar o filtro de papel no suporte.

Passo 4 -> Adicionar a quantidade desejada de pó de café
           dentro do filtro.

Passo 5 -> Despejar lentamente a água quente sobre o
           pó de café.

Passo 6 -> Aguardar o café passar pelo filtro.

Passo 7 -> Servir o café na xícara.


------------------ SAÍDA DE DADOS -------------------

Resultado final:
Uma xícara de café pronta para consumo.


=====================================================

Observação:
Esse exemplo representa um algoritmo do cotidiano,
pois segue uma sequência lógica de instruções
para atingir um objetivo específico.

Analogia:
Como criar uma estrada alternativa ligada à
estrada principal para testar novos caminhos.

+----------------------------------------------------+
| 1 - QUAL É A PRINCIPAL FUNÇÃO DO GIT EM UM PROJETO |
|     DE SOFTWARE?                                   |
+----------------------------------------------------+

a) Armazenar apenas imagens do projeto  
b) Controlar versões e histórico de alterações dos arquivos  
c) Executar códigos automaticamente  
d) Criar interfaces gráficas  

> Resposta correta:
-> b) Controlar versões e histórico de alterações dos arquivos

+----------------------------------------------------+
| 2 - QUAL A DIFERENÇA ENTRE UM REPOSITÓRIO LOCAL E  |
|     UM REPOSITÓRIO REMOTO?                         |
+----------------------------------------------------+

a) O local fica no computador do desenvolvedor e o remoto fica na internet  
b) O remoto só armazena imagens  
c) O local serve apenas para backup  
d) Não existe diferença entre eles  

> Resposta correta:
-> a) O local fica no computador do desenvolvedor e o remoto fica na internet


======================================================


+----------------------------------------------------+
| 3 - O QUE ACONTECE QUANDO FAZEMOS UM COMMIT NO GIT?|

> Resposta correta:
-> b) Uma alteração é registrada no histórico do repositório local

d) Pull → Push → Add

-> c) Add → Commit → Push
